## Project Overview

**WishCraft** — a Vue 3 SPA for shift-wish entry. Displays a month-based calendar where employees express shift preferences (preferred / unavailable / limited) per date, with multi-date drag selection, infinite scroll, and undo/redo. Wishes are saved to a PHP backend on the WishCraft server.

The app is launched via an invite link generated by **ShiftCraft** (the planner tool). The link encodes `planId`, `personId`, `month`, `token`, and optionally a `shifts` fallback. A richer plan config (shift times, holidays) is fetched from the server at startup via `GET /api/load-plan-config.php`.

The full IO specification (invite link format, token model, API endpoints, file structure) is in `docs/IO_SPECS.md`.

## Architecture Notes

- Path alias `@/*` resolves to `./src/*` (configured in both `tsconfig.app.json` and `vite.config.ts`)
- Undo/redo uses `useRefHistory(data, { deep: true, capacity: 50 })` from VueUse — **pause history during init/load** to avoid spurious entries

### Key composables

| File | Responsibility |
|---|---|
| `src/composables/useWishStore.ts` | Wish state: load, merge, save |
| `src/composables/useWishConfig.ts` | `fetchPlanConfig(planId)` — loads shift types + holidays from server |
| `src/composables/useHolidays.ts` | `getGermanFederalHolidays(year)` — fallback holiday calculation |
| `src/utils/urlParams.ts` | `parseContextParams()`, `isSafeId()`, `isYearMonth()`, `buildInviteLink()` |
| `src/utils/token.ts` | `cacheToken()`, `getCachedToken()` — localStorage token persistence |

### PHP backend (`api/`)

| File | Responsibility |
|---|---|
| `WishHandler.php` | Core wish save/load logic; token auth via `plandata/{planId}/{personId}.auth` |
| `ConfigHandler.php` | Plan config save/load logic; `plandata/{planId}/config.json` |
| `save-wishes.php` | POST endpoint — Turnstile + WishHandler |
| `load-wishes.php` | GET endpoint — returns wishes for all or one month |
| `save-plan-config.php` | POST endpoint — ShiftCraft pushes plan config (no Turnstile) |
| `load-plan-config.php` | GET endpoint — WishCraft fetches plan config (no auth) |

### Server file layout

```
plandata/
  .htaccess                        ← Require all denied
  {planId}/
    config.json                    ← Plan-Konfiguration (ShiftCraft-Push)
    {personId}.auth                ← Token-Hash (per Person+Plan, monatsübergreifend)

wishdata/
  .htaccess                        ← Require all denied
  {YYYY-MM}/{planId}/{personId}.json   ← Wunschdaten (kein tokenHash)
```

### ID validation

`isSafeId(s)` accepts `/^[a-zA-Z0-9_-]{1,64}$/` — broader than UUID v4, supports short IDs like `p1` or `anna` from ShiftCraft seed data. All path-dangerous characters (`.`, `/`, spaces) are rejected.
